# 智能体管理后台设计方案

## 功能模块

### skills管理

> 该模块负责管理智能体的技能，包括添加、删除、修改技能等操作。技能表包含字段： id,name,description,file_path,create_time
- **技能列表** 显示所有技能的列表，包括技能名称、描述、文件路径、关联的智能体ID列表、创建时间等信息。

- **上传技能** 上传符合openclaw规范的 .zip 文件，上传后会自动解压到 .uploads/skills目录下，并从skill.md中解析出名称和描述，然后将该记录保存到数据库的 技能表中

- **删除技能** 删除指定技能，同时从数据库中删除该技能的记录，以及从 .uploads/skills目录下删除该技能的所有文件。删除时需要检查该技能是否被关联到任何智能体，如果有关联，需要先解关联该技能，才能删除。

### 智能体管理

> 该模块负责管理智能体的列表，包括添加、删除、修改智能体等操作。智能体表包含字段： id,name,description,prompt,create_time

- **智能体列表** 显示所有智能体的列表，包括智能体名称、描述、提示、创建时间等信息。

- **添加智能体** 添加新的智能体，包括名称、描述、提示词等信息。

- **删除智能体** 删除指定智能体，同时从数据库中删除该智能体的记录。

## 数据库设计

> 使用postgresql数据库，agent 表, skill 表, skill_agent 关联表

## 使用技术

> 前后端分离架构。后端：python, fastapi, postgresql, sqlalchemy

## 后端包管理

> 使用 uv 管理后端依赖
